拉起原生应用

在开发某些应用时可能希望能够调用iOS系统内置的电话、短信、邮件、浏览器应用，此时你可以直接使用UIApplication的OpenURL:方法指定特定的协议来打开不同的系统应用。常用的协议如下：

打电话：tel:或者tel://、telprompt:或telprompt://(拨打电话前有提示)

发短信：sms:或者sms://

发送邮件：mailto:或者mailto://

启动浏览器：http:或者http://

 
//打电话

    NSString *url=[NSString stringWithFormat:@"telprompt://%@", phoneNumber];//这种方式会提示用户确认是否拨打电话
    [self openUrl:url];

-(void)openUrl:(NSString *)urlStr{
    //注意url中包含协议名称，iOS根据协议确定调用哪个应用，例如发送短信是“sms://”其中“//”可以省略写成“sms:”(其他协议也是如此)
    NSURL *url=[NSURL URLWithString:urlStr];
    UIApplication *application=[UIApplication sharedApplication];
    if(![application canOpenURL:url]){
        NSLog(@"无法打开\"%@\"，请确保此应用已经正确安装.",url);
        return;
    }
    [[UIApplication sharedApplication] openURL:url];
}

接受其他进程的参数

就像调用系统应用一样，协议后面可以传递一些参数（例如上面传递的myparams），这样一来在应用中可以在AppDelegate的-(BOOL)application:(UIApplication *)application openURL:(NSURL *)url sourceApplication:(NSString *)sourceApplication annotation:(id)annotation代理方法中接收参数并解析。

 
-(BOOL)application:(UIApplication *)application openURL:(NSURL *)url sourceApplication:(NSString *)sourceApplication annotation:(id)annotation{
    NSString *str=[NSString stringWithFormat:@"url:%@,source application:%@,params:%@",url,sourceApplication,[url host]];
    NSLog(@"%@",str);
    return YES;//是否打开
}

//二维码就是把字符串按照一定的规则转换为了图片
//字符串越长 图片越复杂
NSString *msg = @"http://www.baidu.com";
//1. 字符串转Data 二进制
NSData *data = [msg dataUsingEncoding:NSUTF8StringEncoding];

CIFilter *filter = [CIFilter filterWithName:@"CIQRCodeGenerator"];
//把要修改的元数据放入过滤器
[filter setValue:data forKey:@"inputMessage"];
//获取过滤之后的结果
CIImage *ciIMG = filter.outputImage;
UIImage *img = [[UIImage alloc] initWithCIImage:ciIMG];

扫描二维码

开始扫描

 
//扫描二维码的逻辑
//1.打开后置摄像头
//1.1 获取后置摄像头的控制权, 指针
AVCaptureDevice *device = [AVCaptureDevice defaultDeviceWithMediaType:AVMediaTypeVideo];
//2.读取后置摄像头传入的数据流, 输入流
NSError *error = nil;
AVCaptureDeviceInput *input = [AVCaptureDeviceInput deviceInputWithDevice:device error:&error];
if (error) {
    NSLog(@"%@", error);
    return;
}
//3.需要一条管道, 把输入流导入到屏幕上
AVCaptureMetadataOutput *output = [AVCaptureMetadataOutput new];
_session = [AVCaptureSession new];
//设置管道的传输质量
_session.sessionPreset = AVCaptureSessionPresetHigh;
//4.导入屏幕的数据流, 输入流
[_session addInput:input];
[_session addOutput:output];
//5.输入数据的同时, 不断检测是否有二维码存在. 一旦检测到. 则通过代理通知我们
[output setMetadataObjectsDelegate:self queue:dispatch_get_main_queue()];
//设置要监听的数据类型. 此行代码必须在管道搭建完毕后编写
output.metadataObjectTypes = @[AVMetadataObjectTypeQRCode, AVMetadataObjectTypeEAN13Code, AVMetadataObjectTypeEAN8Code, AVMetadataObjectTypeCode128Code];
​
//把图像显示在页面上
_layer = [AVCaptureVideoPreviewLayer layerWithSession:_session];
//显示内容的区域大小
_layer.frame = self.view.bounds;
//设置内容模式为 保持比例填满
_layer.videoGravity = AVLayerVideoGravityResizeAspectFill;
[self.view.layer addSublayer:_layer];
//启动管道
[_session startRunning];

扫描到结果

通过AVCaptureMetadataOutputObjectsDelegate代理中的方法, 来进行扫描到结果后的操作.

 
- (void)captureOutput:(AVCaptureOutput *)captureOutput didOutputMetadataObjects:(NSArray *)metadataObjects fromConnection:(AVCaptureConnection *)connection{
    if (metadataObjects.count > 0) {
        AVMetadataMachineReadableCodeObject *obj = metadataObjects.firstObject;
        NSString *code  = obj.stringValue;
        NSLog(@"%@", code);
        [_session stopRunning];
        [_layer removeFromSuperlayer];
        //判断扫描到的二维码是不是可以进行应用跳转
        NSURL *url = [NSURL URLWithString:code];
        if ([[UIApplication sharedApplication] canOpenURL:url]) {
            [[UIApplication sharedApplication] openURL:url];
        }
    }
}

推送

参考资料

推送, 专业名字是 APNs. 全称Apple Push Notification service. 即苹果推送通知服务.

在以往的开发中, 都是客户端主动发送请求, 向服务端获取某些数据.

而推送服务正好相反, 是服务端主动联系客户端.

简单说下这个步骤:

  1. 购买开发者资格, 成为Apple的开发者. 这需要$99或者700RMB. 使用visa, 万事达信用卡支付. 通常一天内可以审核通过. 购买地址
  2. 根据参考资料中的教程, 向苹果申请你的应用程序的各种证书. 包括开发证书, 发布证书. 推送证书(开发版), 推送证书(发布版).
  3. 在程序启动时, 向用户申请推送许可. 只有用户允许, 才能进行推送.
  4. 如果用户同意接收推送服务, 则程序会自动向苹果服务器申请一个唯一标识, 称为deviceToken, 这是一个32位的16进制字符串. 这个字符串就是你的这款设备在苹果服务器中的唯一标识. 苹果服务器可以通过此标识找到这款设备. 此字符串会在一个代理方法中传递给你.
  5. 通过服务器给的某个接口, 把这个标识传递给服务器. 这样服务器就知道了你的这个设备和它对应的这个唯一标识.
  6. (这个步骤跟客户端没关系)当服务器想要给客户端推送消息时. 只要拿着这个标识, 外加证书, 就可以要求苹果服务器向对应的客户端发布信息了. 证书是确保服务器的合法性.
  7. 苹果服务器会把消息传递给客户端. 如果客户端是开启在前台的状态. 则会在某个代理方法中接受到这个通知. 如果客户端在后台, 则会有屏幕上的一些提示. 例如弹出框, 声音, 桌面图标数字.
